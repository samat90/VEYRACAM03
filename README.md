# VeyraCam

Веб-приложение для анализа состояния пользователя по видеопотоку с камеры. Работает в реальном времени, без облачных сервисов, все вычисления выполняются локально.

## Возможности

- **Осанка** — угол позвоночника от вертикали или угол линии плеч; калибровка «нормы» под пользователя
- **Моргания** — частота и PERCLOS (процент времени с закрытыми глазами), адаптивный порог EAR
- **Дыхание** — частота циклов по движению плеч относительно головы, bandpass-фильтр 0.1–0.5 Гц
- **Пульс** — rPPG по ROI лба, метод CHROM, частотная область
- **Эмоции** — нейросетевая классификация (EfficientNet-B0, AffectNet), 5 классов
- **Ориентация головы** — pitch/yaw/roll через solvePnP, оценка внимания
- **Зевки** — детекция по Mouth Aspect Ratio
- **Индекс усталости** — композитный показатель 0–100 (PERCLOS, зевки, длинные моргания, поза головы)
- **Рекомендации** — правила с учётом трендов и длительности текущего состояния
- **История сессий** — графики метрик во времени (SQLite + Chart.js)
- **Итоги сессии** — агрегированная сводка после остановки камеры

## Запуск

Требуется Python 3.12 (с 3.13+ сборки `mediapipe 0.10` могут быть недоступны).

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate     # Linux/macOS

pip install -r requirements.txt

python backend/cv_processor/download_models.py

cp .env.example .env            # можно не менять в dev

python backend/manage.py migrate
python backend/manage.py runserver
```

Открыть http://127.0.0.1:8000 в браузере. Для работы камеры нужен HTTPS либо `localhost` (браузер не даст доступ к камере с других IP без сертификата).

## Как это работает

1. Браузер захватывает кадры через WebRTC и отправляет на сервер как base64 JPEG.
2. Сервер прогоняет каждый кадр через три детектора (поза, лицо, дыхание) и возвращает JSON с метриками и кадр с отрисованным скелетом.
3. Раз в 5 секунд метрики сохраняются в SQLite. На каждый N-й кадр вычисляется рекомендация.
4. Frontend рендерит метрики, оверлей скелета, прогресс-бар калибровки и индекс усталости.

## API

| Метод | URL                                   | Описание                             |
| ----- | ------------------------------------- | ------------------------------------ |
| GET   | `/`                                   | Главная страница                     |
| GET   | `/history/`                           | Страница истории сессий              |
| POST  | `/process-frame/`                     | Кадр → JSON с метриками              |
| POST  | `/stop-session/`                      | Завершить сессию, получить сводку    |
| GET   | `/api/sessions/`                      | Список последних 30 сессий           |
| GET   | `/api/history/?session_id=<id>`       | Сэмплы конкретной сессии             |

## Калибровка

При запуске камеры первые ~1.5 секунды идёт калибровка:
- **Осанка**: записывается 30 кадров «нормальной» позы, берётся медиана угла.
- **Моргания**: записывается ~4 секунды, порог EAR вычисляется как `mean(open) - 3.5·std(open)`.

Во время калибровки нужно сидеть ровно и смотреть в камеру.
